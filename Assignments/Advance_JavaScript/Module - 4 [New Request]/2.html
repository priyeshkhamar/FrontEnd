A JavaScript promise is an object that reflects the eventual success or failure of an asynchronous action.
It is a method of handling asynchronous actions in a more structured and understandable manner.

  • Promises exist in three states:
  • Pending : The initial condition, in which neither fulfillment nor rejection occurs.
  • Fulfilled : means that the operation was completed successfully.
  • Rejected : indicates that the operation failed.

    • When a promise is created, it is in the pending status. When the asynchronous action finishes, the promise is either fulfilled or denied.
    If the operation is successful, the promise is fulfilled, and the outcome is provided to the promise's then() callback.
    If the operation fails, the promise is denied, and the cause of the failure is sent to the promise's catch() function.
